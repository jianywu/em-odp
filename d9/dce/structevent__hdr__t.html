◆ state

ev_hdr_state_t event_hdr_t::state

Event state, updated on valid state trasitions. "Best effort" update, i.e. atomic update not guaranteed in invalid simultaneous state-updates.

Contains the previously known good state and will be printed when detecting an invalid state transition.

◆ state_cnt

evstate_cnt_t event_hdr_t::state_cnt

Together the evstate_cnt_t counters (evgen, free_cnt and send_cnt) can be used to detect invalid states and operations on the event, e.g.: double-free, double-send, send-after-free, free-after-send, usage-after-output, usage-after-timer-tmo-set/ack/cancel/delete etc.
